데이터베이스(DB) 기초

1. DB란 무엇인가?

데이터베이스(Database)는 여러 사람이 공유하여 사용할 목적으로 통합하여 관리되는 데이터의 집합입니다.

  • 중복성 최소화: 파일 시스템의 단점인 데이터 중복을 방지
  • 무결성 유지: 데이터의 정확성과 일관성 보장
  • DBMS: 이런 DB를 관리하는 소프트웨어 (예: MySQL, Oracle)

DB의 기본 아키텍처

애플리케이션(클라이언트)은 직접 데이터를 조작하지 않고, DBMS를 통해 데이터를 저장하고 조회합니다.

2. RDBMS (관계형 DB)

Relational Database Management System

데이터를 행(Row)과 열(Column)로 이루어진 테이블(표) 형태로 저장하고, 테이블 간의 관계를 정의하는 가장 전통적이고 널리 쓰이는 DB입니다.

a. 테이블 (Table)

  • Column (열/속성): 데이터의 종류 (예: 이름, 나이, 이메일)
  • Row (행/레코드): 실제 데이터의 한 건 (예: 홍길동, 25, hong@...)
  • Primary Key (기본키): 각 행을 고유하게 식별하는 값
  • Foreign Key (외래키): 다른 테이블과 연결고리가 되는 값

b. SQL (구조화된 질의어)

RDBMS와 대화하기 위한 표준 프로그래밍 언어입니다.

  • DDL (정의어): 테이블 생성, 수정, 삭제 (CREATE, ALTER, DROP)
  • DML (조작어): 데이터 삽입, 조회, 수정, 삭제 (INSERT, SELECT, UPDATE, DELETE)
  • DCL (제어어): 권한 부여 및 회수 (GRANT, REVOKE)

SELECT 이름, 나이 
FROM 사용자 
WHERE 나이 >= 20;

c. 트랜잭션과 ACID (핵심 추가)

데이터베이스의 상태를 변화시키는 하나의 논리적 작업 단위입니다. RDBMS의 가장 강력한 무기입니다.

  • A (Atomicity, 원자성): 모두 성공하거나 모두 실패해야 함 (All or Nothing)
  • C (Consistency, 일관성): 작업 전후로 데이터 무결성이 유지되어야 함
  • I (Isolation, 고립성): 동시에 실행되는 트랜잭션은 서로 영향을 주지 않음
  • D (Durability, 지속성): 성공한 결과는 영구적으로 반영됨

d. 정규화 (Normalization)

데이터의 중복을 최소화하고 이상 현상(Anomaly)을 방지하기 위해 데이터를 구조화하는 과정입니다.

  • 제1정규형: 모든 컬럼의 값은 원자값(단일값)을 가져야 함
  • 제2정규형: 부분 함수 종속 제거 (기본키의 일부에만 종속되는 컬럼 분리)
  • 제3정규형: 이행적 함수 종속 제거 (일반 컬럼끼리 종속되는 것 분리)

* 무조건 쪼개는 것이 답은 아니며, 성능을 위해 '반정규화'를 하기도 합니다.

e. ERD (Entity-Relationship Diagram)

데이터베이스의 뼈대(설계도)를 시각적으로 표현한 다이어그램입니다.
요구사항을 분석하여 개체(Entity)와 그들 간의 관계(Relationship)를 그립니다.

3. NoSQL

Not Only SQL / Non-Relational

RDBMS의 한계(유연성 부족, 수평적 확장 어려움)를 극복하기 위해 등장한 다양한 형태의 비관계형 데이터베이스입니다.

a. 문서형 (Document)

데이터를 JSON, XML 같은 문서 형태로 저장합니다. 스키마가 자유롭습니다.

  • 대표 DB: MongoDB, CouchDB

b. Key-Value

단순한 키(Key)와 값(Value)의 쌍으로 데이터를 저장합니다. 속도가 매우 빠릅니다.

  • 대표 DB: Redis, Amazon DynamoDB

c. 와이드 컬럼 (Wide-Column)

행마다 컬럼의 이름과 개수가 다를 수 있는 구조입니다. 대량의 데이터 분석과 분산 처리에 유리합니다.

  • 대표 DB: Cassandra, HBase

d. 그래프 (Graph)

데이터를 노드(Node)와 간선(Edge)으로 연결하여 '관계' 자체를 저장하고 탐색하는 데 특화되어 있습니다. (예: SNS 친구 추천)

  • 대표 DB: Neo4j

4. 그 외의 특수 목적 DB

기술이 발전하면서 특정 목적에 최적화된 DB들이 각광받고 있습니다.

a. 시계열 DB (Time-Series)

시간의 흐름에 따라 연속적으로 생성되는 데이터를 처리하는 데 특화되어 있습니다.

  • 활용: 주식 데이터, IoT 센서 데이터, 서버 모니터링 로그
  • 대표 DB: InfluxDB, Prometheus

b. 벡터 DB (Vector DB)

최근 AI, LLM 시대에 가장 핫한 DB입니다. 비정형 데이터(텍스트, 이미지)를 숫자 배열(벡터)로 변환해 저장하고 유사도 검색을 수행합니다.

  • 활용: AI 챗봇의 문맥 검색, 이미지 유사도 검색
  • 대표 DB: Pinecone, Milvus, Chroma

c. 인메모리 DB (In-Memory)

데이터를 디스크(HDD/SSD)가 아닌 주기억장치(RAM)에 저장하여 압도적인 입출력 속도를 제공합니다.

  • 활용: 캐싱(Caching), 실시간 리더보드, 세션 관리
  • 대표 DB: Redis, Memcached

5. 클라우드 DB란?

AWS, GCP, Azure 같은 클라우드 서비스 제공자가 구축하고 관리해주는 데이터베이스 서비스(DBaaS)입니다.

클라우드 DB의 장점

  • 관리의 편의성: 백업, 패치, 복구 등을 클라우드 벤더가 알아서 해줍니다.
  • 확장성(Scalability): 클릭 몇 번으로 스토리지나 성능을 늘릴 수 있습니다.
  • 고가용성: 데이터 센터에 문제가 생겨도 다른 지역(Region)에서 즉시 대체 동작합니다.
  • 대표 서비스: Amazon RDS, Aurora, Google Cloud SQL

결론

세상에 완벽하고 유일한 데이터베이스는 없습니다.

정확성과 무결성이 중요하면 RDBMS를,
유연한 스키마와 빠른 확장이 필요하면 NoSQL을,
AI나 캐싱 등 특수한 목적이 있다면 그에 맞는 특화 DB를 선택하는 것이 올바른 아키텍처 설계입니다.
